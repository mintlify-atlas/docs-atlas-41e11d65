---
title: "Configure MongoDB for Mydva user authentication"
description: "Connect Mydva's Express.js auth service to MongoDB. Covers Atlas setup, local MongoDB, connection string format, PORT, and JWT_SECRET configuration."
---

Mydva's Express.js authentication server (`mybackend/`) uses MongoDB to persist user data. You need a running MongoDB instance and a valid connection URI before the auth server will start successfully. MongoDB Atlas is recommended for most deployments because it requires no local installation and provides a generous free tier.

## What MongoDB stores

The auth service writes one document per registered user to the configured database. Each document contains:

| Field | Description |
|---|---|
| `FullName` | The user's display name, provided at registration |
| `email` | Unique email address used for login |
| `password` | Bcrypt-hashed password — the plaintext password is never stored |
| `profile` | Filename of the uploaded profile photo (stored in `public/images/`) |

## Get a MongoDB connection URI

### MongoDB Atlas (recommended)

MongoDB Atlas is a fully managed cloud database with a free M0 tier suitable for development and small production workloads.

<Steps>
  <Step title="Create an Atlas account">
    Go to [atlas.mongodb.com](https://atlas.mongodb.com) and sign up for a free account.
  </Step>
  <Step title="Create a cluster">
    In the Atlas dashboard, click **Build a Database** and select the **Free** (M0) tier. Choose a cloud provider and region closest to your server.
  </Step>
  <Step title="Create a database user">
    Under **Database Access**, add a new database user with a username and a strong password. Note both — you will need them in the connection string.
  </Step>
  <Step title="Allow network access">
    Under **Network Access**, add your server's IP address to the IP Access List. For local development you can add `0.0.0.0/0` to allow all IPs, but restrict this in production.
  </Step>
  <Step title="Copy the connection string">
    In the cluster view, click **Connect**, choose **Drivers**, and copy the connection string. It will look like:

    ```
    mongodb+srv://username:password@cluster0.example.mongodb.net/?retryWrites=true&w=majority
    ```

    Replace `username` and `password` with the credentials from the previous step, and append your database name before the `?`:

    ```
    mongodb+srv://username:password@cluster0.example.mongodb.net/mydva?retryWrites=true&w=majority
    ```
  </Step>
</Steps>

### Local MongoDB

If you prefer a local installation, install MongoDB Community Edition and use the following URI format:

```
mongodb://localhost:27017/mydva
```

No username or password is required for a default local installation.

## Add the URI to your environment

Open or create `DVA/mybackend/.env` and set the following three variables:

```env
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/mydva
PORT=5000
JWT_SECRET=your_jwt_secret
```

### PORT

The `PORT` variable sets the port the Express auth server listens on. The default is `5000`. If another service is already using port `5000` on your machine — such as the Flask chatbot, which also defaults to port `5000` — change this value to an available port and update any frontend configuration that references it.

### JWT_SECRET

The `JWT_SECRET` is the key used to sign JSON Web Tokens issued at login. Any string works, but it must be long and random to be secure. To generate a strong secret, run one of the following commands:

<CodeGroup>

```bash Linux / macOS
openssl rand -base64 48
```

```bash Windows (PowerShell)
[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 }))
```

</CodeGroup>

Paste the output as the value of `JWT_SECRET`. Do not reuse this value across environments.

<Warning>
If you change `JWT_SECRET` after users have logged in, all existing sessions will be invalidated and users will need to log in again.
</Warning>
