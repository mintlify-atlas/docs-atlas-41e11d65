Mydva runs four services locally: a FastAPI backend for dosha analysis, an Express.js backend for authentication, a Flask chatbot, and a React frontend. This guide walks you through cloning the repository, configuring your environment, starting all services, and completing your first consultation.

## Prerequisites

Before you begin, make sure you have the following installed and available:

- **Python 3.10 or later** — for the FastAPI backend and Flask chatbot
- **Node.js 18 or later** — for the Express.js auth backend and React frontend
- **Git** — to clone the repository
- **Groq API key** — obtain one for free at [console.groq.com](https://console.groq.com)
- **MongoDB URI** — a connection string for a MongoDB database (Atlas free tier or local instance)

## Set up the project

<Note>
  Mydva requires **two separate `.env` files**: one in the project root (for the FastAPI backend) and one in `mybackend/` (for the Express.js auth backend). Both must be configured before starting any service.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/GaurRitika/DVA
    cd DVA
    ```
  </Step>

  <Step title="Create a Python virtual environment">
    From the `DVA/` root directory, create and activate a virtual environment:

    <CodeGroup>

    ```bash Linux / macOS
    python3 -m venv venv
    source venv/bin/activate
    ```

    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    ```

    </CodeGroup>
  </Step>

  <Step title="Install Python dependencies">
    With the virtual environment active, install the required packages from the project root:

    ```bash
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Configure environment variables">
    Create the two required `.env` files.

    **Root `.env`** (used by the FastAPI backend and React frontend):

    ```bash .env
    GROQ_API_KEY=your_groq_api_key_here
    REACT_APP_API_BASE_URL=http://localhost:8000/api/v1
    ```

    **`mybackend/.env`** (used by the Express.js auth backend):

    ```bash mybackend/.env
    MONGO_URI=your_mongodb_connection_string
    JWT_SECRET=your_jwt_secret_here
    PORT=5000
    ```

    <Tip>
      Generate a strong `JWT_SECRET` with `openssl rand -hex 32` on Linux/macOS or any secure random string generator.
    </Tip>
  </Step>

  <Step title="Start the FastAPI backend">
    Open a new terminal, navigate to the `backend/` folder, activate the virtual environment, then start the server:

    <CodeGroup>

    ```bash Linux / macOS
    cd backend
    source ../venv/bin/activate
    uvicorn app:app --reload --host 0.0.0.0 --port 8000
    ```

    ```bash Windows
    cd backend
    ..\venv\Scripts\activate
    uvicorn app:app --reload --host 0.0.0.0 --port 8000
    ```

    </CodeGroup>

    The API will be available at `http://localhost:8000`.
  </Step>

  <Step title="Start the Express.js auth backend">
    Open a new terminal and run:

    ```bash
    cd mybackend
    npm install
    npm run dev
    ```

    The auth service listens on port `5000`.
  </Step>

  <Step title="Start the Flask chatbot">
    Open a new terminal, activate the virtual environment from the `DVA/` root, then start the chatbot service:

    <CodeGroup>

    ```bash Linux / macOS
    source venv/bin/activate
    cd mybackend/final_bot
    python main.py
    ```

    ```bash Windows
    venv\Scripts\activate
    cd mybackend\final_bot
    python main.py
    ```

    </CodeGroup>

    Flask starts on port `5000` by default. If the Express backend is already on `5000`, set a different port:

    ```bash
    FLASK_RUN_PORT=5001 python main.py
    ```
  </Step>

  <Step title="Start the React frontend">
    Open a new terminal and run:

    ```bash
    cd frontend-react
    npm install
    npm start
    ```

    The app opens at `http://localhost:3000`.
  </Step>
</Steps>

## Your first consultation

With all four services running, complete the following steps in your browser:

1. **Register an account** — navigate to `http://localhost:3000` and click **Sign Up**. Enter your name, email, and password. The Express.js backend stores your credentials in MongoDB with a hashed password and issues a JWT.

2. **Take the dosha quiz** — after logging in, select **Dosha Analysis** from the navigation. Answer the questionnaire about your physical constitution, digestion, and temperament. Submit the form to send your responses to the FastAPI backend at `/api/v1/analyze-dosha`.

3. **View your results** — the API returns your primary dosha type (Vata, Pitta, or Kapha) along with a breakdown. Your results are displayed on the dashboard and stored against your account.

4. **Get a personal consultation** — from the results page, click **Get Consultation**. Mydva sends your dosha profile to the Groq LLaMA 3.3 70B model via `/api/v1/personal-consultation` and returns a detailed, personalised Ayurvedic guidance report.

5. **Chat with the assistant** — use the chatbot widget to ask follow-up questions. The Flask chatbot service, powered by Google Gemini, answers context-aware Ayurvedic queries at any time.
