---
title: "GET /api/v1/recommendations/{dosha_type}"
description: "GET /api/v1/recommendations/{dosha_type} — fetch AI Ayurvedic guidance for vata, pitta, or kapha. Returns diet, lifestyle, herbal, and exercise sections."
---

The `/api/v1/recommendations/{dosha_type}` endpoint retrieves a structured set of Ayurvedic recommendations for a given dosha type. Unlike the `/analyze-dosha` endpoint, this route does not require quiz input — it returns general guidance directly keyed to the specified dosha. The recommendations are generated by an AI model and cover diet, lifestyle adjustments, herbal remedies, therapeutic practices, and exercise.

## Endpoint

```
GET http://localhost:8000/api/v1/recommendations/{dosha_type}
```

## Path parameters

<ParamField path="dosha_type" type="string" required>
  The dosha type for which to fetch recommendations. Must be one of the following values:

  - `vata`
  - `pitta`
  - `kapha`

  Any other value will return an error response.
</ParamField>

## Example requests

<CodeGroup>

```bash vata
curl --request GET \
  --url http://localhost:8000/api/v1/recommendations/vata
```

```bash pitta
curl --request GET \
  --url http://localhost:8000/api/v1/recommendations/pitta
```

```bash kapha
curl --request GET \
  --url http://localhost:8000/api/v1/recommendations/kapha
```

</CodeGroup>

## Response

### 200 — success

<ResponseField name="status" type="string" required>
  Always `"success"` on a successful response.
</ResponseField>

<ResponseField name="data" type="object" required>
  A structured recommendations object.

  <Expandable title="data properties" defaultOpen>
    <ResponseField name="dietary" type="string[]" required>
      Food and nutrition guidance aligned with the dosha.
    </ResponseField>

    <ResponseField name="lifestyle" type="string[]" required>
      Daily routine and behavioural practices that support dosha balance.
    </ResponseField>

    <ResponseField name="herbal" type="string[]" required>
      Herbs, supplements, and Ayurvedic formulations recommended for this dosha.
    </ResponseField>

    <ResponseField name="therapeutic" type="string[]" required>
      Ayurvedic therapies such as Panchakarma treatments and oil therapies.
    </ResponseField>

    <ResponseField name="exercise" type="string[]" required>
      Exercise types and intensities suited to the dosha constitution.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="error" type="null">
  Always `null` on a successful response.
</ResponseField>

```json

  "status": "error",
  "data": null,
  "error": "Invalid dosha type. Must be one of: vata, pitta, kapha."

```

<Note>
Recommendations are generated per request by an AI model and may vary slightly between calls. Cache the response on the client side if consistent output across a session is important.
</Note>
